=== 2. DATABASE (Supabase Postgres, run in SQL Editor) ===
- profiles table: id (uuid, references auth.users), is_premium (boolean, default false), premium_until (timestamptz, nullable), sakhi_message_count (int, default 0), created_at (timestamptz, default now()). Auto-create via trigger on auth.users insert. Enable RLS: users SELECT their own row; users UPDATE only their own is_premium (needed for client-side RevenueCat sync).
- live_sessions table: share_id (uuid, primary key, default gen_random_uuid()), user_id (uuid, references auth.users), lat/lng/accuracy (double precision), is_active (boolean, default true), started_at/updated_at (timestamptz, default now()), expires_at (timestamptz, default now() + interval '6 hours'). Enable RLS: owning user can do anything on their own rows (using/with check auth.uid() = user_id); anyone holding the share_id can SELECT rows where is_active = true and expires_at > now() (this is the public read policy the contact's web page uses). Enable Realtime replication on this table via Dashboard > Database > Replication.

=== 3. BACKEND (Node/Express server hosted in this Repl, NOT Supabase Edge Functions) ===
Create server/index.js — one Express server with CORS + JSON body parsing, reading SUPABASE_URL, SUPABASE_SERVICE_ROLE_KEY, OPENAI_API_KEY, GOOGLE_PLACES_API_KEY, REVENUECAT_WEBHOOK_SECRET from Replit Secrets (never hardcoded). Listen on process.env.PORT, bind 0.0.0.0. Add a .replit file: run = "node server/index.js".

Routes:
a) POST /sakhi-chat — verify the caller's Supabase auth token (Authorization: Bearer <token>) via supabase.auth.getUser(token) using the service role key. Look up their profile. If NOT premium (is_premium false, or premium_until expired) AND sakhi_message_count >= 5, return 402 { error: "limit_reached", message: "...upgrade to Premium..." }. Otherwise forward the conversation to OpenAI's /v1/chat/completions (model "gpt-4o-mini", system prompt establishing "Sakhi" as a warm safety companion that defers to calling 112 in emergencies), return the reply, increment sakhi_message_count for free users only. Never store chat messages themselves.

b) POST /revenuecat-webhook — verify Authorization Bearer header against REVENUECAT_WEBHOOK_SECRET. On INITIAL_PURCHASE/RENEWAL/UNCANCELLATION/PRODUCT_CHANGE, set is_premium=true and premium_until from expiration_at_ms. On EXPIRATION, set is_premium=false. (Inactive until I'm on RevenueCat's Pro plan, since webhooks aren't on their free tier — return 501 if REVENUECAT_WEBHOOK_SECRET isn't set.)

c) GET /nearby-places?lat=&lng=&type=police|hospital|pharmacy|shelter — proxies Google Places "Nearby Search (New)" (searchText for "shelter" since there's no direct Google type, searchNearby for the rest) using X-Goog-Api-Key and a strict X-Goog-FieldMask limited to id, displayName, formattedAddress, location only — do not add photos/rating/openingHours, those push billing into pricier tiers. Add a simple 5-minute in-memory cache keyed by category+rounded lat/lng to cut duplicate calls. Return { places: [{ id, name, address, lat, lng }] }.
