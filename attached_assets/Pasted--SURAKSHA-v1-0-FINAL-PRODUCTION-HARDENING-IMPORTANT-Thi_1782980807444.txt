# SURAKSHA v1.0 FINAL PRODUCTION HARDENING

IMPORTANT

This is the final production hardening task before Play Store and App Store release.

DO NOT redesign the UI.

DO NOT recreate any screens.

DO NOT modify the visual design.

Preserve all existing functionality.

Only improve architecture, authentication, backend stability, production readiness, performance, and reliability.

==================================================
1. AUTHENTICATION AUDIT
==================================================

Audit the complete authentication system.

Determine whether the application uses Firebase Authentication.

If Firebase Authentication is enabled:

Remove ALL remaining Supabase Authentication code.

Remove:

• signInWithOtp()
• signInWithMagicLink()
• auth.signInWithOtp()
• Supabase email authentication
• Magic Link authentication
• Supabase session handling
• Supabase password reset
• Supabase logout

Firebase must be the ONLY authentication provider.

Use:

• Firebase Email & Password Authentication
• Firebase Phone OTP Authentication
• Firebase Password Reset
• Firebase Session Persistence
• Firebase Sign Out
• Google Sign-In (Android/Web if configured)
• Apple Sign-In (iOS if configured)

Supabase must ONLY be used for:

• Database
• Storage
• Realtime
• Row Level Security

NOT Authentication.

Search the entire project and remove every remaining Supabase auth reference.

==================================================
2. SUPABASE ARCHITECTURE
==================================================

Create ONLY ONE shared Supabase client.

Use ONLY

EXPO_PUBLIC_SUPABASE_URL

EXPO_PUBLIC_SUPABASE_PUBLISHABLE_KEY

Remove

placeholder URLs

placeholder keys

dummy values

fake clients

fallback clients

multiple createClient() calls

legacy initialization

Supabase should initialize only after configuration validation succeeds.

==================================================
3. CONFIGURATION VALIDATION
==================================================

Keep lib/config.ts as the single source of truth.

Validate on startup:

EXPO_PUBLIC_SUPABASE_URL

EXPO_PUBLIC_SUPABASE_PUBLISHABLE_KEY

EXPO_PUBLIC_FIREBASE_API_KEY

EXPO_PUBLIC_FIREBASE_AUTH_DOMAIN

EXPO_PUBLIC_FIREBASE_PROJECT_ID

EXPO_PUBLIC_FIREBASE_STORAGE_BUCKET

EXPO_PUBLIC_FIREBASE_MESSAGING_SENDER_ID

EXPO_PUBLIC_FIREBASE_APP_ID

If ANY variable is missing:

Do NOT initialize Firebase.

Do NOT initialize Supabase.

Do NOT crash.

Display ConfigErrorScreen.

Development:

Show exactly which variables are missing.

Production:

Display only

Configuration Error

Some required application services are unavailable.

Please contact support.

Never expose technical information.
