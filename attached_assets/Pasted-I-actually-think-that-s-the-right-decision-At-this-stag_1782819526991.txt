Replit AI Prompt – Focus on Functionality, Not UI

Objective

Do not redesign the UI unless required for functionality. Keep the existing Suraksha interface, colors, spacing, and navigation.

The priority is to make every feature fully functional, reliable, production-ready, and usable during real emergencies.

Every feature should work end-to-end without mock data.

⸻

Priority 1 – One-Tap SOS (Highest Priority)

Replace the current demo SOS flow with a real emergency workflow.

When the user taps and holds the SOS button:

* Immediately capture GPS location.
* Start a 3-second countdown with vibration.
* Automatically send SMS to all trusted contacts.
* Automatically share a Google Maps live location link.
* Automatically open WhatsApp with a pre-filled emergency message and location.
* Automatically call the first trusted contact.
* If unanswered within 20 seconds, proceed to the next trusted contact.
* If no trusted contact answers, offer to call 112 immediately.
* Continue updating the user’s location every 15 seconds while SOS mode is active.
* Display delivery and sending status for every action.
* Allow cancellation only with biometric authentication or a 3-second hold to prevent accidental cancellation.

⸻

Priority 2 – Trusted Contacts

Implement complete contact management.

* Add/Edit/Delete contacts.
* Import directly from the phone’s contacts.
* Validate phone numbers.
* Support multiple emergency contacts.
* Store securely in Supabase/Firebase.
* Sync across devices.
* Mark a primary emergency contact.

⸻

Priority 3 – SMS Integration

Implement real SMS sending.

Emergency message example:

“🚨 EMERGENCY

Varun may be in danger.

Live Location:
https://maps.google.com/?q=LAT,LNG

Please contact immediately.”

Include:

* Name
* Time
* GPS coordinates
* Google Maps link
* Battery percentage (optional)

⸻

Priority 4 – Live Location Tracking

During SOS or Journey Mode:

* Update GPS every 10–15 seconds.
* Store locations in the backend.
* Share live tracking with trusted contacts.
* Stop automatically when the user ends SOS.

⸻

Priority 5 – Journey Timer

When a journey starts:

* Ask for destination.
* Let the user choose duration or ETA.
* Share the trip with trusted contacts.
* Notify contacts when the journey starts.
* If the timer expires without confirmation:
    * Trigger SOS automatically.
    * Send the latest location.
    * Notify contacts.

⸻

Priority 6 – Community Reports

Replace placeholders with a working system.

Allow users to:

* Report unsafe areas.
* Upload photos.
* Select incident category.
* Add comments.
* View reports on the map.
* Upvote or confirm reports.
* Automatically expire reports after a configurable time.

⸻

Priority 7 – Safety Map

Implement:

* Nearby police stations.
* Hospitals.
* Pharmacies.
* Emergency shelters.
* Community reports.
* Heatmap for unsafe locations.
* Real route navigation.

Priority 8 – Fake Call

Generate a realistic incoming call.

Support:

Priority 9 – Weather Intelligence

Use a weather API to:

* Show severe weather alerts.
* Display flood, rain, heatwave, and thunderstorm warnings.
* Update the Safety Score based on weather conditions.
* Recommend safety actions.

⸻

Priority 10 – Notifications

Implement push notifications for:

* SOS activation.
* Journey started.
* Journey completed.
* Missed check-in.
* Community alerts.
* Severe weather alerts.
* Safety reminders.

⸻

Priority 11 – Authentication

Support:

* Phone OTP.
* Email login.
* Google Sign-In.
* Apple Sign-In (iOS).
* Persistent sessions.
* Password reset.
* Secure logout.

⸻

Priority 12 – Offline Mode

When the internet is unavailable:

* Continue GPS tracking.
* Queue SMS requests.
* Cache emergency contacts.
* Cache legal rights.
* Sync automatically when connectivity returns.

⸻

Priority 13 – AI Assistant (Sakhi)

Implement an AI assistant that can:

* Answer safety questions.
* Explain women’s legal rights.
* Suggest nearby safe places.
* Guide users during emergencies.
* Recommend actions based on time, weather, and location.

⸻

Priority 14 – Backend

Replace all mock data with real backend functionality.

Use Supabase/Firebase for:

* Authentication.
* Contacts.
* Community reports.
* Live locations.
* Journey history.
* User preferences.
* Premium subscriptions.

⸻

Priority 15 – Error Handling

Never expose technical messages such as:

* “Auth session missing”
* “Unsupported phone provider”
* “Network error”

Show friendly messages and log detailed errors internally.

⸻

Final Goal

Every feature in Suraksha should function reliably in a real emergency. Prioritize stability, speed, offline resilience, and user safety over visual changes. Ensure all emergency workflows are tested, use real integrations where possible, and eliminate all placeholder or simulated behavior.
